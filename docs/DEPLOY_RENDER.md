# API Deployment — Render

## Prerequisites

- Render account at [render.com](https://render.com)
- GitHub repo connected to Render
- Supabase project already running with schema applied
- All required third-party keys ready (Stripe, Mercado Pago, Resend)

---

## Step 1 — Connect repo to Render

1. Render Dashboard → **New +** → **Web Service**
2. Connect GitHub → select this repo
3. Render will detect `render.yaml` automatically and pre-fill settings

If it doesn't auto-detect, set manually:
| Field | Value |
|---|---|
| **Root Directory** | `.` (repo root, not `apps/api`) |
| **Runtime** | Node |
| **Build Command** | `npm install -g pnpm && pnpm install --frozen-lockfile && pnpm --filter @pablo/api build` |
| **Start Command** | `node apps/api/dist/index.js` |
| **Health Check Path** | `/health` |

> Root must be repo root (not `apps/api`) — pnpm needs the full monorepo to resolve workspace packages.

---

## Step 2 — Set environment variables

In Render dashboard → **Environment** tab, add these:

### Required

| Variable | Where to get it |
|---|---|
| `SUPABASE_URL` | Supabase Dashboard → Project Settings → API → Project URL |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase Dashboard → Project Settings → API → service_role key |
| `SESSION_SECRET` | Auto-generated by Render (`generateValue: true` in render.yaml) |
| `CORS_ORIGIN` | Comma-separated list of allowed frontend URLs, e.g. `https://admin.yourdomain.com,https://yourapp.com` |

### Stripe

| Variable | Where to get it |
|---|---|
| `STRIPE_SECRET_KEY` | Stripe Dashboard → Developers → API keys → Secret key |
| `STRIPE_WEBHOOK_SECRET` | Stripe Dashboard → Developers → Webhooks → signing secret |

### Mercado Pago

| Variable | Where to get it |
|---|---|
| `MP_ACCESS_TOKEN` | MP Developer Dashboard → Credentials → Access Token |
| `MP_WEBHOOK_SECRET` | MP Developer Dashboard → Webhooks → secret |

### Email (Resend)

| Variable | Value |
|---|---|
| `RESEND_API_KEY` | Resend Dashboard → API Keys |
| `EMAIL_FROM` | Verified sender email, e.g. `noreply@yourdomain.com` |

### Optional

| Variable | Notes |
|---|---|
| `SENTRY_DSN` | Sentry project DSN for error tracking. Skip if not using Sentry. |
| `B2_KEY_ID` | Backblaze B2 — only needed for backup scripts |
| `B2_APPLICATION_KEY` | Backblaze B2 |
| `B2_BUCKET_NAME` | Backblaze B2 |

### Auto-set by render.yaml (do not change)

```
NODE_ENV=production
PORT=4000
HOST=0.0.0.0
LOG_LEVEL=info
```

---

## Step 3 — Deploy

Click **Create Web Service**. Render runs the build command and starts the server.

Monitor logs in Render Dashboard → **Logs** tab. Successful start looks like:

```
API running on 0.0.0.0:4000
```

Verify with the health check endpoint:

```
https://your-service.onrender.com/health
```

Response should be:

```json
{ "ok": true, "ts": "2026-01-01T00:00:00.000Z" }
```

---

## Step 4 — Update frontend env vars

After deploy, copy your Render service URL (e.g. `https://pablo-api.onrender.com`).

**Vercel (admin)** — add/update env var:
```
NEXT_PUBLIC_API_URL=https://pablo-api.onrender.com
```

Redeploy Vercel admin app for the change to take effect.

---

## Step 5 — Configure webhooks

### Stripe

1. Stripe Dashboard → Developers → Webhooks → Add endpoint
2. URL: `https://pablo-api.onrender.com/api/webhooks/stripe`
3. Events: select all relevant events (payment_intent, charge, etc.)
4. Copy the signing secret → set as `STRIPE_WEBHOOK_SECRET` in Render

### Mercado Pago

1. MP Developer Dashboard → Webhooks
2. URL: `https://pablo-api.onrender.com/api/webhooks/mercadopago`
3. Copy secret → set as `MP_WEBHOOK_SECRET` in Render

---

## Free plan caveats

Render free plan **spins down after 15 minutes of inactivity**. First request after spin-down takes ~30s.

To avoid cold starts, either:
- Upgrade to Render **Starter** plan ($7/mo) — always on
- Use an uptime monitor (e.g. UptimeRobot) to ping `/health` every 10 minutes

---

## Switching to client's Render account

When ready to hand off:

1. Client creates Render account and connects the GitHub repo
2. Client creates new Web Service with same settings above
3. Client sets all env vars (share the list from Step 2)
4. Update `NEXT_PUBLIC_API_URL` in Vercel to point to client's Render URL
5. Update webhook URLs in Stripe and Mercado Pago
6. Delete the service on our Render account
